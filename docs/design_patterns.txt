Design Patterns in UniConnect

This document shows the main design patterns we used in this project.


1. Singleton Pattern
We used this in src/services/api.ts by exporting one single api object. This keeps all our backend calls consistent in one place.


2. Observer Pattern
We used this in App.tsx with the event listener for notifications. It lets different parts of the app talk to each other without being strictly linked.


3. Component Pattern
All our UI like Badge, Modal, and Avatar are built as reusable components. This keeps the design looking the same across all pages.


4. Layout Pattern
The AppShell in App.tsx wraps every page with the same Sidebar and TopBar.


5. Strategy Pattern
The pageLabels object in the TopBar helps pick the right title based on which page you are visiting.


6. Polling Pattern
We use setInterval in App.tsx and LostFoundPage.tsx to check for new messages or items every few seconds so it feels live.


7. State Lifting
We keep the main user and page info in App.tsx and pass it down to the other pages. This keeps the "truth" in one central spot.
