The Technology Stack Fundamentals

Every tool, language, and framework used in UniConnect — what it is, what it does, and why we chose it.

⚛

React & TypeScript

Frontend Architecture

React builds your app out of small, reusable "components" (like a button, a card, or an entire page). When data changes, React re-renders only the affected parts. We pair this with TypeScript to strictly type all data shapes (e.g., ensuring a User always has an email and name). This catches bugs before they even hit the browser.

Why we use it: Modular scaling for a massive 17-page application, with rock-solid type safety.
⚡

Vite

Build Tool & Dev Server

Vite powers our local development. It runs a local web server with Hot Module Replacement (HMR) to instantly refresh your browser when you save a file. For production, Vite optimally bundles all React code into lightweight, minified assets that run fast in any browser.

Why we use it: Lightning-fast compilation. Changes appear in <100ms.
Ex

Express.js & Node

Backend Server

Node.js runs Javascript outside the browser. Express is our minimal framework on top of Node. It intercepts HTTP requests from the frontend, queries our database, and returns the response as JSON. It serves as the bridge between your UI and your persistent data.

Why we use it: One language (JavaScript) used uniformly across the entire stack.
🗄

PostgreSQL & SQLite

Dual Database Architecture

We use a dual-database approach via a custom abstraction layer. Locally, we use SQLite for instant, zero-config development. In production on Vercel, the app automatically seamlessly switches to PostgreSQL via connection pooling. The same SQL queries execute natively on both.

Why we use it: The best of both worlds. Zero friction locally, but robust scalability for live deployment on Vercel.
🔑

JWT (JSON Web Tokens)

Authentication System

JWT handles login sessions statelessly. When a user logs in, the server creates a signed token containing their ID. This token is stored in the browser's localStorage and passed along with every API call. If the token is valid, the server processes the request.

Why we use it: Stateless authentication means our server doesn't have to remember who's logged in, drastically reducing memory usage and making Vercel serverless deployment easy.
☁️

Vercel

Cloud Deployment Host

Vercel is the cloud platform that hosts the live UniConnect platform. It serves the static React frontend from a global CDN and executes our Express backend API dynamically via serverless functions (configured in vercel.json). It also provides the cloud PostgreSQL database.

Why we use it: Auto-scaling, free SSL, CI/CD out of the box, and native Postgres support.

main.tsx & App.tsx Master Entry

main.tsx is the ignition key mounting the React DOM. App.tsx is the orchestrator managing currentUser state and dictating which view (Dashboard, Messages, Login, etc.) to show based on standard React state updates.

Backend Files & DB Abstraction server/

The Express API with our new universal Database layer.

The Database Abstraction Layer

db-connection.js Database Layer

The secret sauce for Vercel deployment. Reads process.env.DB_TYPE. If it equals postgres, it boots up the pg Pool and creates an adapter that perfectly mimics SQLite's .get(), .run(), and .all() API. This means zero routing logic has to change when moving to the cloud.

schema.sql Schema Definitions

A unified SQL file defining all 12 tables. The system reads this file dynamically on boot to ensure the database (whether SQLite or Postgres) has the correct table structures.

db.js Legacy Export Bridge

Our endpoints still `require('./db')`. This file now just imports from `db-connection.js` and waits for it to initialize, exporting standard database verbs to keep backwards compatibility.

Migration & Safety Scripts

migrate-data.js Script

Reads every single row from your local `uniconnect.db` SQLite file and meticulously inserts it into the active PostgreSQL connection. Ensures zero data loss when moving from local dev to Vercel production.

rollback-migration.js Script

A safety valve. Safely truncates and cascades data deletions in the PostgreSQL database if a migration errors out or needs to be wiped clean.

Express Routing Logic

index.js Server Entry

Mounts all routes (auth, users, partners, groups, resources, messages, etc) under /api. Initializes middleware to verify JWTs.

The Database Schema SQL Tables

The 12 tables structuring the UniConnect ecosystem, fully compatible with Postgres & SQLite.

Users

  • id TEXT PK
  • name TEXT NOT NULL
  • email TEXT UNIQUE
  • passwordHash TEXT
  • role DEFAULT 'student'
  • avgRating REAL DEFAULT 0

PartnerRequests

  • id TEXT PK
  • fromId → Users(id)
  • toId → Users(id)
  • status DEFAULT 'pending'

StudyGroups

  • id TEXT PK
  • name TEXT NOT NULL
  • creatorId → Users(id)

GroupMembers

  • id TEXT PK
  • userId → Users(id)
  • groupId → StudyGroups(id)
  • UNIQUE(userId, groupId)

Resources

  • id TEXT PK
  • title TEXT NOT NULL
  • status DEFAULT 'available'
  • ownerId → Users(id)

BorrowRequests & Transactions

  • BorrowRequests:
  • resourceId → Resources(id)
  • status DEFAULT 'pending'
  • Transactions:
  • status DEFAULT 'active'
  • rating INTEGER

Conversations & Messages

  • convId TEXT PK
  • user1Id / user2Id → Users(id)
  • Messages:
  • senderId → Users(id)
  • text TEXT

Notifications & LostFound

  • Notifications:
  • content / type TEXT
  • LostFound:
  • status DEFAULT 'open'
  • reporterId → Users(id)

Configuration & Deployment Files Root

The new files added to manage Vercel deployments and dynamic configurations.

vercel.json Deployment

Instructs Vercel precisely how to build and host our app. It maps /api/(.*) to our Express Node.js environment via @vercel/node, while serving our built Vite assets from the root. Sets the default DB_TYPE environment variable for production to postgres.

server/.env.example & server/.env.production Env Setup

We rely on `.env` files to dictate the database mode. Locally, `DB_TYPE=sqlite` skips Postgres usage. In Vercel, `DB_TYPE=postgres` is used alongside the auto-injected `DATABASE_URL`.

package.json Config

Now contains custom `npm run migrate` and `npm run rollback` commands to trigger our custom database bridging scripts.

Glossary of Terms Reference

Quick definitions for terms you'll encounter throughout the codebase.

TermDefinition
ServerlessVercel's approach to Node apps: The Express backend boots up in milliseconds only when an API call is made, and shuts down immediately, saving cloud costs.
Connection PoolingBecause Vercel serverless functions boot rapidly, we use `pg-pool` to manage the connections to Postgres safely so the database isn't overwhelmed.
ComponentA reusable piece of UI in React. Can be as small as a button or as large as an entire page.
StateData that changes over time within a component. Managed with useState(). React auto-updates when state changes.
REST APIAn architectural style for APIs. Uses GET (read), POST (create), PUT (update), DELETE (remove).
HMRHot Module Replacement. Vite updates only the changed module in the browser without a full reload.
JWTAn encoded string containing user data signed with a secret key. Proves identity statelessly.
